1. STMS-specific mission variable

In this repository, the mission variable is not abstract stability or general control performance. It is thermal feasibility preservation: a modeled thermal state remains within allowable margin while useful throughput is retained despite radiative load, storage/buffer saturation, power limitation, sensing delay, and degraded actuator authority. The question is whether the hybrid separated-barrier route delays or avoids thermal-margin collapse in the reduced surrogate, not whether a flight system has been built.

2. Reduced thermal-control surrogate

A10-STMS should be read as a nondimensional reduced-surrogate audit of hybrid thermal control. It does not define radiator hardware, thermal straps, spacecraft layout, thermal-vacuum qualification, flight control code, or operational procedure. The useful object is the thermal-feasibility diagnostic map.

STMS-specific surrogate reading

  • The state channels are thermal-control diagnostics: thermal margin, throughput, storage, power, delay, and actuator authority.
  • The model is reduced and nondimensional; it is not a spacecraft thermal design tool.
  • The value is in stress-channel separation and claim-bounded feasibility screening.
  • Hardware-in-the-loop, flight qualification, and thermal-vacuum validation are outside scope.

3. STMS stress channels

The differentiating feature of A10-STMS is that failure has thermal-control meanings: thermal-margin collapse, throughput loss, storage saturation, power-capped correction, delay-induced overshoot, actuator-authority loss, or combined thermal-stress failure.

Why this differs from BMS/RPT

  • BMS failure is SOC / voltage / thermal / plating feasibility.
  • RPT failure is pressure-thermal-supply regime collapse.
  • STMS failure is thermal-margin and throughput collapse under cooling-resource and delay constraints.
  • The mission variable is therefore STMS-specific, not a generic A10 placeholder.

4. Separated-barrier interpretation

A project-specific strength of A10-STMS is that thermal failure channels are separated: radiative loading, storage saturation, power limitation, sensing delay, and actuator degradation are not forced into one opaque scalar. The structured route applies barrier-like protection to distinct thermal failure modes.

Diagnostic interpretation

  • A power barrier cannot by itself solve sensing delay.
  • A storage barrier cannot by itself solve actuator-authority loss.
  • Separated barriers make failure modes visible instead of hiding them in one metric.
  • Barrier success in a surrogate does not imply flight qualification.

5. Hybrid separated-barrier control loop

The A10 prior in this project restricts the thermal-control response space so that radiative, storage, power, delay, and actuator constraints can be separated and stress-tested. It is not a claim that A10 replaces spacecraft thermal engineering or validates a cooling product.
